Voraussetzung als root

Podman installieren

apt update && apt install -y podman
dnf install -y podman

RHEL 9 / Rocky / Alma: vorher dnf module enable container-tools:rhel9 -y ausführen.

pacman -S podman
apk add podman crun
1

YAML-Datei speichern

sudo mkdir -p /etc/containers/
# YAML oben kopieren nach:
sudo nano /etc/containers/{{ pod_name }}.yaml
2

Pod testen (ohne systemd)

sudo podman play kube /etc/containers/{{ pod_name }}.yaml

# Status:
sudo podman pod ps && sudo podman ps

# Stoppen:
sudo podman play kube --down /etc/containers/{{ pod_name }}.yaml
3

Quadlet .kube-Datei erstellen (systemd)

Datei ablegen unter /etc/containers/systemd/{{ pod_name }}.kube

sudo mkdir -p /etc/containers/systemd/
cat << 'EOF' | sudo tee /etc/containers/systemd/{{ pod_name }}.kube
[Unit]
Description={{ pod_name }} Pod

[Kube]
Yaml=/etc/containers/{{ pod_name }}.yaml

[Install]
WantedBy=multi-user.target
EOF
4

systemd aktivieren (rootful)

sudo systemctl daemon-reload
sudo systemctl enable --now {{ pod_name }}-pod.service

Status & Logs

sudo systemctl status {{ pod_name }}-pod.service
sudo journalctl -u {{ pod_name }}-pod.service -f
sudo podman pod ps
sudo podman ps

OpenRC-Service einrichten

Alpine nutzt OpenRC — kein Quadlet. Init-Script als root anlegen:

cat > /etc/init.d/{{ pod_name }}-pod << 'INITEOF'
#!/sbin/openrc-run
name="{{ pod_name }}-pod"
description="{{ pod_name }} Pod (Podman rootful)"
command="/usr/bin/podman"
command_args="play kube /etc/containers/{{ pod_name }}.yaml"
command_background=true
pidfile="/run/${RC_SVCNAME}.pid"

depend() {
    need net cgroups
}

stop() {
    ebegin "Stopping ${name}"
    start-stop-daemon --stop --pidfile "${pidfile}"
    podman play kube --down /etc/containers/{{ pod_name }}.yaml
    eend $?
}
INITEOF
chmod +x /etc/init.d/{{ pod_name }}-pod
rc-update add cgroups boot
rc-update add {{ pod_name }}-pod default
rc-service {{ pod_name }}-pod start
4

Status & Logs

rc-service {{ pod_name }}-pod status
podman pod ps && podman ps
Alpine / OpenRC:
  • Kein systemd → kein Quadlet. Der Pod wird direkt per podman play kube über OpenRC gestartet.
  • Logs: podman logs <container-name> oder /var/log/messages
Rootful-Betrieb:
  • Container laufen als root — nur für vertrauenswürdige Images verwenden
  • Ports < 1024 direkt bindbar
  • Kein loginctl enable-linger nötig — systemd verwaltet den Service
  • Quadlet-Pfad: /etc/containers/systemd/ (nicht ~/.config/)
