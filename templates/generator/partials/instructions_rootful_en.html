Install Podman

apt update && apt install -y podman
dnf install -y podman

Save the YAML file

sudo mkdir -p /etc/containers/
# Copy the YAML above to:

Test the pod (without systemd)

sudo podman play kube /etc/containers/{{ pod_name }}.yaml

# Status:
sudo podman pod ps && sudo podman ps

# Stop:
sudo podman play kube --down /etc/containers/{{ pod_name }}.yaml

Create Quadlet .kube file (systemd)

Copy the {{ pod_name }}.kube file from the tab above and save it to /etc/containers/systemd/:

sudo mkdir -p /etc/containers/systemd/

Set up OpenRC service

Alpine uses OpenRC — no Quadlet. Create the init script as root:

cat > /etc/init.d/{{ pod_name }}-pod << 'INITEOF'
#!/sbin/openrc-run
name="{{ pod_name }}-pod"
description="{{ pod_name }} Pod (Podman rootful)"
command="/usr/bin/podman"
command_args="play kube /etc/containers/{{ pod_name }}.yaml"
command_background=true
pidfile="/run/${RC_SVCNAME}.pid"

depend() {
    need net cgroups
}

stop() {
    ebegin "Stopping ${name}"
    start-stop-daemon --stop --pidfile "${pidfile}"
    podman play kube --down /etc/containers/{{ pod_name }}.yaml
    eend $?
}
INITEOF
chmod +x /etc/init.d/{{ pod_name }}-pod
rc-update add cgroups boot
rc-update add {{ pod_name }}-pod default
rc-service {{ pod_name }}-pod start
4

Status & Logs

rc-service {{ pod_name }}-pod status
podman pod ps && podman ps
Alpine / OpenRC:
  • No systemd → no Quadlet. The pod is started directly via podman play kube through OpenRC.
  • Logs: podman logs <container-name> or /var/log/messages
Rootful mode:
  • Containers run as root — only use trusted images
  • Ports < 1024 can be bound directly
  • No loginctl enable-linger needed — systemd manages the service
  • Quadlet path: /etc/containers/systemd/ (not ~/.config/)
