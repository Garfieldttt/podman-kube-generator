Prerequisites once as root

0. Install Podman

apt update && apt install -y podman
dnf install -y podman

RHEL 9 / Rocky / Alma: run dnf module enable container-tools:rhel9 -y first.

pacman -S podman
apk add podman crun

1. Create user (if not existing)

useradd -m -s /bin/bash {{ deploy_user }}
passwd {{ deploy_user }}

1. Create user (if not existing)

adduser -D -s /bin/sh {{ deploy_user }}
passwd {{ deploy_user }}

2. Enable linger (service runs after reboot without login)

loginctl enable-linger {{ deploy_user }}

2. Set up OpenRC service (replaces loginctl)

Alpine uses OpenRC — no systemd, no loginctl. Create an init script:

cat > /etc/init.d/{{ pod_name }}-pod << 'INITEOF'
#!/sbin/openrc-run
name="{{ pod_name }}-pod"
description="{{ pod_name }} Pod (Podman rootless)"
command_user="{{ deploy_user }}"
command="/usr/bin/podman"
command_args="play kube /home/{{ deploy_user }}/.config/containers/{{ pod_name }}.yaml"
command_background=true
pidfile="/run/${RC_SVCNAME}.pid"

depend() {
    need net cgroups
}

stop() {
    ebegin "Stopping ${name}"
    start-stop-daemon --stop --pidfile "${pidfile}"
    su - {{ deploy_user }} -c "podman play kube --down /home/{{ deploy_user }}/.config/containers/{{ pod_name }}.yaml"
    eend $?
}
INITEOF
chmod +x /etc/init.d/{{ pod_name }}-pod
rc-update add cgroups boot
rc-update add {{ pod_name }}-pod default

Enable automatic image updates

Requires AutoUpdate=registry in the .kube file. Podman then checks for new images and restarts the pod automatically:

systemctl --user start podman.socket
systemctl --user daemon-reload
systemctl --user enable --now podman-auto-update.timer

# Check status:
systemctl --user status podman-auto-update.timer

# Trigger manually:
podman auto-update
3

Apply image updates

rc-service {{ pod_name }}-pod restart
Alpine / OpenRC:
  • No systemd → no Quadlet. The pod is started directly via podman play kube through OpenRC.
  • Logs: podman logs <container-name> or /var/log/messages
