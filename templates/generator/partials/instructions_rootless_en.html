Prerequisites once as root

0. Install Podman

apt update && apt install -y podman
dnf install -y podman

RHEL 9 / Rocky / Alma: run dnf module enable container-tools:rhel9 -y first.

pacman -S podman
apk add podman crun
brew install podman
podman machine init
podman machine start

Alternatively: installer package from podman.io. Podman Desktop (optional) manages the machine automatically at login.

1. Create user (if not existing)

useradd -m -s /bin/bash {{ deploy_user }}
passwd {{ deploy_user }}

1. Create user (if not existing)

adduser -D -s /bin/sh {{ deploy_user }}
passwd {{ deploy_user }}

On macOS, Podman runs inside a VM — no separate system user required. Continue as your normal user.

2. Enable linger (service runs after reboot without login)

loginctl enable-linger {{ deploy_user }}

2. Set up OpenRC service (replaces loginctl)

Alpine uses OpenRC — no systemd, no loginctl. Create an init script:

cat > /etc/init.d/{{ pod_name }}-pod << 'INITEOF'
#!/sbin/openrc-run
name="{{ pod_name }}-pod"
description="{{ pod_name }} Pod (Podman rootless)"
command_user="{{ deploy_user }}"
command="/usr/bin/podman"
command_args="play kube /home/{{ deploy_user }}/.config/containers/{{ pod_name }}.yaml"
command_background=true
pidfile="/run/${RC_SVCNAME}.pid"

depend() {
    need net cgroups
}

stop() {
    ebegin "Stopping ${name}"
    start-stop-daemon --stop --pidfile "${pidfile}"
    su - {{ deploy_user }} -c "podman play kube --down /home/{{ deploy_user }}/.config/containers/{{ pod_name }}.yaml"
    eend $?
}
INITEOF
chmod +x /etc/init.d/{{ pod_name }}-pod
rc-update add cgroups boot
rc-update add {{ pod_name }}-pod default

No root steps required on macOS. Podman Machine and autostart are configured as a normal user (see Step 3 below).

3

Set up autostart via launchd

Create a LaunchAgent for the current user. The Podman Machine must already be running at login (e.g. via Podman Desktop).

mkdir -p ~/Library/LaunchAgents/
cat > ~/Library/LaunchAgents/com.podman.{{ pod_name }}.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.podman.{{ pod_name }}</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/podman</string>
    <string>play</string>
    <string>kube</string>
    <string>/Users/YOURUSER/.config/containers/{{ pod_name }}.yaml</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/tmp/{{ pod_name }}-pod.log</string>
  <key>StandardErrorPath</key>
  <string>/tmp/{{ pod_name }}-pod.log</string>
</dict>
</plist>
EOF
launchctl load ~/Library/LaunchAgents/com.podman.{{ pod_name }}.plist
4

Status & Logs

launchctl list | grep com.podman.{{ pod_name }}
podman pod ps && podman ps
cat /tmp/{{ pod_name }}-pod.log
5

Apply image updates

podman pull docker.io/<image>:<tag>
launchctl stop com.podman.{{ pod_name }}
launchctl start com.podman.{{ pod_name }}
macOS:
  • Podman runs inside a Linux VM (Podman Machine) — Quadlet files on the macOS host are not supported.
  • Path to podman: Homebrew Apple Silicon → /opt/homebrew/bin/podman, Intel → /usr/local/bin/podman
  • Replace YOURUSER in the plist with your macOS username (echo $USER).
