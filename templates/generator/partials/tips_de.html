Nützliche Tipps

Ports < 1024 (z.B. 80, 443)

Rootless darf keine privilegierten Ports öffnen. Lösung:

sysctl net.ipv4.ip_unprivileged_port_start=80

Persistent in /etc/sysctl.d/99-podman.conf eintragen.

Container kommunizieren über localhost

Alle Container im Pod teilen sich den Netzwerk-Namespace. Verbindungen immer über localhost, nicht über Container-Namen.

# Richtig (z.B. App → DB):
localhost:5432

# Falsch (funktioniert im Pod nicht):
db-container:5432

Offene Ports anzeigen

Welche Ports hört der laufende Pod?

podman port {{ pod_name }}-pod

DNS im Pod

Eigenen DNS-Server für den Pod setzen (z.B. lokales Pi-hole):

# In der YAML unter spec.dnsConfig:
spec:
  dnsConfig:
    nameservers:
      - 192.168.1.x

Volume-Besitzer setzen

Bei Permission-Fehlern UID/GID im User-Namespace anpassen:

podman unshare chown 1000:1000 /pfad/zum/volume

SELinux Volume-Labels

Bei SELinux-Systemen (RHEL, Fedora) Volume-Suffix setzen:

/host/path:/container/path:Z   # privat
/host/path:/container/path:z   # geteilt

Alle Volumes auflisten

podman volume ls
podman volume inspect <volume-name>

Volume-Backup

Daten aus einem Volume sichern:

podman run --rm \
  -v <volume-name>:/data:ro \
  -v $(pwd):/backup \
  busybox tar czf /backup/backup.tar.gz /data

Aufräumen

Nicht mehr verwendete Images, Container und Volumes entfernen:

podman system prune -f        # Container + Images
podman image prune -f         # nur ungetaggte Images
podman volume prune -f        # ungenutzte Volumes

Automatische Image-Updates (podman-auto-update)

Podman kann Images automatisch aktualisieren und den Pod neu starten. Einmalig aktivieren:

systemctl --user start podman.socket
systemctl --user daemon-reload
systemctl --user enable --now podman-auto-update.timer
systemctl --user status podman-auto-update.timer

Testen ohne wirklich zu updaten:

podman auto-update --dry-run

Manuelles Update

Einzelnes Image aktualisieren und Pod neu starten:

podman pull <image>:<tag>
podman play kube --replace \
  ~/.config/containers/{{ pod_name }}.yaml

Veraltete Images finden

Lokale Images mit Remote vergleichen:

podman images --filter dangling=false
podman pull --all-tags <image>

Shell in laufenden Container

podman exec -it {{ pod_name }}-<container> /bin/sh
# oder bash:
podman exec -it {{ pod_name }}-<container> /bin/bash

Logs live verfolgen

# Alle Container im Pod:
podman pod logs -f {{ pod_name }}-pod

# Einzelner Container:
podman logs -f {{ pod_name }}-<container>

Pod-Infos & Ressourcen

podman pod inspect {{ pod_name }}-pod
podman stats {{ pod_name }}-pod

Pod neu starten ohne Datenverlust

podman pod restart {{ pod_name }}-pod

# oder via systemd:
systemctl --user restart {{ pod_name }}-pod.service
