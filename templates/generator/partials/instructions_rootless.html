Voraussetzungen einmalig als root

0. Podman installieren

apt update && apt install -y podman
dnf install -y podman

RHEL 9 / Rocky / Alma: vorher dnf module enable container-tools:rhel9 -y ausführen.

pacman -S podman
apk add podman crun
brew install podman
podman machine init
podman machine start

Alternativ: Installer-Paket von podman.io. Podman Desktop (optional) verwaltet die Machine automatisch beim Login.

1. User anlegen (falls nicht vorhanden)

useradd -m -s /bin/bash {{ pod_name }}
passwd {{ pod_name }}

1. User anlegen (falls nicht vorhanden)

adduser -D -s /bin/sh {{ pod_name }}
passwd {{ pod_name }}

Auf macOS läuft Podman in einer VM — kein separater System-User nötig. Als normaler User weiterarbeiten.

2. subuid/subgid einrichten

usermod --add-subuids 100000-165535 --add-subgids 100000-165535 {{ pod_name }}

2. subuid/subgid

Fedora richtet subuid/subgid beim Anlegen neuer User automatisch ein.

Für RHEL / Rocky / Alma oder bestehende User: usermod --add-subuids 100000-165535 --add-subgids 100000-165535 {{ pod_name }}

2. subuid/subgid

Ab shadow ≥ 4.11.1-3 (Standard seit 2022) werden subuid/subgid automatisch angelegt.

Bei linux-hardened-Kernel zusätzlich: sysctl kernel.unprivileged_userns_clone=1 (dauerhaft in /etc/sysctl.d/).

2. subuid/subgid einrichten

echo "{{ pod_name }}:100000:65536" >> /etc/subuid
echo "{{ pod_name }}:100000:65536" >> /etc/subgid

3. Linger aktivieren (Service läuft nach Reboot ohne Login)

loginctl enable-linger {{ pod_name }}

3. OpenRC-Service einrichten (Autostart statt loginctl)

Alpine nutzt OpenRC — kein systemd, kein loginctl. Init-Script erstellen:

cat > /etc/init.d/{{ pod_name }}-pod << 'INITEOF'
#!/sbin/openrc-run
name="{{ pod_name }}-pod"
description="{{ pod_name }} Pod (Podman rootless)"
command_user="{{ pod_name }}"
command="/usr/bin/podman"
command_args="play kube /home/{{ pod_name }}/.config/containers/{{ pod_name }}.yaml"
command_background=true
pidfile="/run/${RC_SVCNAME}.pid"

depend() {
    need net cgroups
}

stop() {
    ebegin "Stopping ${name}"
    start-stop-daemon --stop --pidfile "${pidfile}"
    su - {{ pod_name }} -c "podman play kube --down /home/{{ pod_name }}/.config/containers/{{ pod_name }}.yaml"
    eend $?
}
INITEOF
chmod +x /etc/init.d/{{ pod_name }}-pod
rc-update add cgroups boot
rc-update add {{ pod_name }}-pod default

Auf macOS sind keine Root-Schritte nötig. Podman Machine und Autostart werden als normaler User eingerichtet (siehe Schritt 3 unten).

1

YAML-Datei speichern

mkdir -p ~/.config/containers/
# YAML oben kopieren nach:
nano ~/.config/containers/{{ pod_name }}.yaml
2

Pod testen (ohne Autostart)

podman play kube ~/.config/containers/{{ pod_name }}.yaml

# Status prüfen:
podman pod ps && podman ps

# Stoppen:
podman play kube --down ~/.config/containers/{{ pod_name }}.yaml
3

Quadlet .kube-Datei erstellen

Ablegen unter ~/.config/containers/systemd/{{ pod_name }}.kube

mkdir -p ~/.config/containers/systemd/
cat > ~/.config/containers/systemd/{{ pod_name }}.kube << 'EOF'
[Unit]
Description={{ pod_name }} Pod

[Kube]
Yaml=%h/.config/containers/{{ pod_name }}.yaml

[Install]
WantedBy=default.target
EOF
4

systemd Service aktivieren

systemctl --user daemon-reload
systemctl --user enable --now {{ pod_name }}-pod.service
5

Status & Logs

systemctl --user status {{ pod_name }}-pod.service
journalctl --user -u {{ pod_name }}-pod.service -f
podman pod ps
podman ps
6

Image-Update einspielen

Neue Image-Versionen laden und Pod neu starten:

podman pull docker.io/<image>:<tag>
podman play kube --replace ~/.config/containers/{{ pod_name }}.yaml

# oder via systemd:
systemctl --user restart {{ pod_name }}-pod.service
3

Service starten

Das Init-Script wurde in den Voraussetzungen angelegt. Service jetzt starten:

rc-service {{ pod_name }}-pod start
4

Status & Logs

rc-service {{ pod_name }}-pod status
podman pod ps && podman ps
5

Image-Update einspielen

podman pull docker.io/<image>:<tag>
rc-service {{ pod_name }}-pod restart
Alpine / OpenRC:
  • Kein systemd → kein Quadlet. Der Pod wird direkt per podman play kube über OpenRC gestartet.
  • Logs: podman logs <container-name> oder /var/log/messages
3

Autostart via launchd einrichten

LaunchAgent für den aktuellen User erstellen. Die Podman Machine muss beim Login bereits laufen (z.B. über Podman Desktop).

mkdir -p ~/Library/LaunchAgents/
cat > ~/Library/LaunchAgents/com.podman.{{ pod_name }}.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.podman.{{ pod_name }}</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/podman</string>
    <string>play</string>
    <string>kube</string>
    <string>/Users/DEINUSER/.config/containers/{{ pod_name }}.yaml</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/tmp/{{ pod_name }}-pod.log</string>
  <key>StandardErrorPath</key>
  <string>/tmp/{{ pod_name }}-pod.log</string>
</dict>
</plist>
EOF
launchctl load ~/Library/LaunchAgents/com.podman.{{ pod_name }}.plist
4

Status & Logs

launchctl list | grep com.podman.{{ pod_name }}
podman pod ps && podman ps
cat /tmp/{{ pod_name }}-pod.log
5

Image-Update einspielen

podman pull docker.io/<image>:<tag>
launchctl stop com.podman.{{ pod_name }}
launchctl start com.podman.{{ pod_name }}
macOS:
  • Podman läuft in einer Linux-VM (Podman Machine) — Quadlet-Dateien auf dem macOS-Host werden nicht unterstützt.
  • Pfad zu podman: Homebrew Apple Silicon → /opt/homebrew/bin/podman, Intel → /usr/local/bin/podman
  • DEINUSER im plist durch deinen macOS-Benutzernamen ersetzen (echo $USER).
